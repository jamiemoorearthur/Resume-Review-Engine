# CV Reviewer — Backend

FastAPI backend for the CV Reviewer application. Accepts a CV (PDF or TXT) and a job description, runs it through a RAG pipeline backed by ChromaDB, and returns structured feedback generated by GPT-4o-mini.

## Tech Stack

| Layer | Technology |
|---|---|
| API framework | FastAPI + Uvicorn |
| LLM | OpenAI GPT-4o-mini |
| Vector store | ChromaDB (persistent) |
| Embeddings | OpenAI text-embedding-3-small |
| PII detection | Microsoft Presidio + spaCy |
| Observability | Langfuse |
| Hosting | Fly.io (always-on, no cold starts) |
| Future hosting | Azure AKS (Kubernetes manifests ready) |

## API Response

`POST /review` returns:

```json
{
  "overall_score": 72,
  "ats_score": 68,
  "recruiter_score": 6,
  "category_scores": {
    "role_alignment": 75,
    "skills_match": 70,
    "experience_relevance": 65,
    "ats_keyword_match": 68,
    "bullet_point_quality": 60,
    "structure_readability": 80,
    "missing_evidence": 55
  },
  "role_alignment": "Good",
  "missing_keywords": ["Docker", "PostgreSQL"],
  "strengths": ["Strong Python experience", "..."],
  "weaknesses": ["No quantified achievements", "..."],
  "section_recommendations": ["Add a Technical Skills section"],
  "suggested_bullets": [
    {
      "original": "Built a backend service",
      "improved": "Built and deployed a FastAPI backend service handling 10k requests/day"
    }
  ]
}
```

**Fields:**
- `overall_score` — weighted average across 7 rubric categories (0–100)
- `ats_score` — ATS keyword match score (0–100)
- `recruiter_score` — likelihood a recruiter shortlists this CV in a 6-second scan (0–10)
- `category_scores` — individual score for each of the 7 rubric categories before weighting
- `section_recommendations` — sections to add or remove based on the role
- `suggested_bullets` — scales dynamically with CV quality: 4–6 rewrites for weak CVs, 0–1 for strong CVs

## Local Development

**Prerequisites:** Python 3.11, an OpenAI API key

```bash
cd backend
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python -c "from spacy.cli import download; download('en_core_web_sm')"
```

Copy `.env.example` to `.env` and fill in your keys:

```
OPENAI_API_KEY=sk-...
LANGFUSE_PUBLIC_KEY=pk-lf-...
LANGFUSE_SECRET_KEY=sk-lf-...
LANGFUSE_HOST=https://cloud.langfuse.com
```

Run the server:

```bash
uvicorn app.main:app --reload
```

API available at `http://localhost:8000`. Docs at `http://localhost:8000/docs`.

## Deployment — Fly.io (current)

The backend is hosted on Fly.io at `https://cv-reviewer-api.fly.dev`.

- Always-on (`min_machines_running = 1`, `auto_stop_machines = false`) — no cold starts
- 512MB RAM, `shared-cpu-1x`, London (`lhr`) region
- ChromaDB data persisted on a 1GB Fly volume mounted at `/app/data/chroma`

**To deploy manually:**

```bash
cd backend
fly deploy
```

**Secrets are set via:**

```bash
fly secrets set OPENAI_API_KEY="..." LANGFUSE_PUBLIC_KEY="..." ...
```

CI auto-deploys to Fly on every push to `main` (requires `FLY_API_TOKEN` GitHub secret).

## Infrastructure — Azure AKS (future migration)

Terraform and Kubernetes manifests are ready for when the product is funded and ready to migrate off Fly.io. All Azure CI/CD workflows are manual-only and will not trigger automatically.

**Terraform** (`terraform/`):
- Provisions AKS cluster, ACR, Key Vault, Log Analytics, Azure File Share (ChromaDB persistence)
- Remote state stored in `cvreviewertfstate` storage account (`cvreviewer-bootstrap` resource group)
- Node size: `Standard_D2s_v5` (x64, 2 vCPU, 8GB RAM)
- Region: UK South

**Kubernetes manifests** (`k8s/`):
- Namespace, Deployment, Service (LoadBalancer), PVC (Azure File Share), SecretProviderClass (Key Vault CSI)

**GitHub Actions workflows** (`.github/workflows/`):

| Workflow | Trigger | Purpose |
|---|---|---|
| `aks-provision.yml` | Manual only | Bootstrap tfstate storage + terraform apply |
| `aks-cd.yml` | Manual only | Build linux/amd64 image, push to ACR, deploy to AKS |
| `aks-destroy.yml` | Manual (`confirm: DESTROY`) | Tear down all Azure resources |

**GitHub secrets needed when migrating:**

| Secret | Where to get it |
|---|---|
| `AZURE_CLIENT_ID` | App registration in Azure AD |
| `AZURE_CLIENT_SECRET` | App registration → Certificates & secrets |
| `AZURE_TENANT_ID` | Azure AD → Overview |
| `AZURE_SUBSCRIPTION_ID` | Azure portal → Subscriptions |
| `ACR_LOGIN_SERVER` | Azure Container Registry → Overview |
