# Strong CV Bullet Point Examples

CV bullets should follow this structure:
**Action Verb + What You Did + Result or Impact (quantified where possible)**

Weak bullets describe duties. Strong bullets describe achievements.

---

## Software Engineering

**Weak:** Responsible for developing backend APIs for the company platform.
**Strong:** Built and deployed 12 RESTful APIs in Python/FastAPI serving 40,000 daily active users, reducing average response time by 35% through query optimisation and Redis caching.

## General Rules for Strong Bullets

1. **Start with a strong action verb** — Built, Designed, Led, Delivered, Reduced, Increased, Automated, Migrated, Deployed, Optimised
2. **Be specific** — name the technology, tool, team size, or time frame
3. **Quantify impact** — use numbers: %, £, time, users, requests/second, team size
4. **One bullet = one achievement** — do not combine multiple accomplishments into one bullet
5. **Avoid passive voice** — "was responsible for" → "led", "built", "owned"
6. **Avoid filler words** — "various", "multiple", "a number of" add no information

## Action Verbs by Category

**Built / Delivered:** Built, Developed, Designed, Architected, Shipped, Deployed, Implemented, Created, Launched

**Improved:** Optimised, Reduced, Improved, Increased, Accelerated, Streamlined, Automated, Refactored

**Led:** Led, Managed, Owned, Directed, Coordinated, Mentored, Trained, Facilitated

**Analysed:** Analysed, Investigated, Audited, Assessed, Evaluated, Modelled, Researched
